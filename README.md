# ML-сервис прогнозирования подписания спецификации

## Описание проекта

Учебный ML-сервис для прогнозирования успешного завершения заказа

p.s. грузила файлами, потому что почему-то не скачивается git(((

Модель предсказывает, дойдёт ли заказ до статуса:

**«Спецификация подписана / заказ закрыт»**

Сервис построен на основе очищенных обезличенных данных из Excel о заказах с начала 2026 года.

Компания выступает как посредник между исполнителями и заказчиками на рынке металлообработки. 
Схема такая:
- приходит заказчик со своим заказом
- компания ищет исполнителей под данный заказ, отбирается один/несколько, они выставляет коммерчское предложение
- на основе окоммерческого предложения исполнителя выставляется коммерческое предложение заказчику

Проект демонстрирует взаимодействие:

```text
Пользователь → Streamlit → FastAPI → ML-модель
```

## Целевая переменная

Используется столбец **«Укрупнённый статус»**:

* 1 — «Спецификация подписана / заказ закрыт»;
* 0 — любой другой статус.

## Признаки модели

Для обучения используются:

* Источник заявки;
* Сегмент;
* Стоимость для заказчика
* Стоимость исполнителя

Для категориальных признаков применяется OneHotEncoder, для числовых — обработка пропусков. В качестве модели используется Random Forest.

## Запуск проекта

Обучение модели:

```bash
python utils/train_model.py
```

Запуск API:

```bash
uvicorn backend.api:app --reload
```

Запуск интерфейса:

```bash
python -m streamlit run frontend/Main.py
```

## Результат

Пользователь вводит параметры заказа через веб-интерфейс, после чего модель оценивает вероятность успешного подписания спецификации и выводит результат на экран
